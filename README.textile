h1. OneBusAway Java API

This library provides a client for "OneBusAway":http://www.onebusaway.org/ API webapp. You find more informations about OneBusAway on the Google Code "project page":http://code.google.com/p/onebusaway/.

h2. Java & Android

This library is fully compatible with Android.

Please ask for enhancements and report bugs on "Github tracker":https://github.com/kops/nominatim-java-api/issues/new or "contact us":http://www.google.com/recaptcha/mailhide/d?k=01Th60_7w3rxWuSJumsnqxfg==&c=REgbsYXndhO58POROxZGybu0F_Xu3JmR-wBRNbh8knE.

h2. Unit testing

To run unit tests, the _URL_ and the _key_ of the test server must be provided. There is two ways to do this:
 
h5. from the command line

Run a command like @mvn test -Donebusaway.api.testUrl=http://server.net/onebusaway-api-webapp/api -Donebusaway.api.testKey=key@.

h5. or define a profile

Add the following in your _settings.xml_ file:

<pre>
    <profiles>
        <profile>
            <id>one-busaway-tests</id>
            <properties>
                <onebusaway.api.testUrl>http://api.itinerennes.fr/onebusaway-api-webapp/api</onebusaway.api.testUrl>
                <onebusaway.api.testKey>web</onebusaway.api.testKey>
            </properties>
        </profile>
    </profiles>
</pre>

Then you have to activate this profile:

<pre>
    <activeProfiles>
        <activeProfile>one-busaway-tests</activeProfile>
    </activeProfiles>
</pre>

Or run @mvn test -Pone-busaway-tests@
